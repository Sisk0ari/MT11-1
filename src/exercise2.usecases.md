# Модуль входа в систему и аутентификации:

### Сценарий 1: Регистрация нового пользователя

Как: Пользователь через форму авторизации

Цель: зарегистрироваться как новый пользователь

Дано: нахожусь в окне авторизации на сайте https://widaxo.rocket.chat/home 

Чтобы зарегистрироваться мне нужно:

- нажать на кнопку "Create an account"
- ввести личные данные
- нажать кнопку "Join your team"
- подтвердить адрес электронной почты
- подтвердить регистрацию, перейдя по специальной ссылке в письме на электронную почту

Результат: регистрация совершена

### Сценарий 2: Авторизация существующего пользователя 

Как: Пользователь через форму авторизации

Цель: авторизоваться на сайте, используя существующую учетную запись

Дано: нахожусь в окне авторизации на сайте https://widaxo.rocket.chat/home 

Чтобы авторизоваться мне нужно:

- ввести данные пользователя (логин и пароль)
- нажать кнопку "Login"

Результат: авторизация совершена

### Сценарий 3: Восстановление пароля

Как: Пользователь через форму авторизации

Цель: восстановить доступ к аккаунту, изменив пароль

Дано: нахожусь в окне авторизации на сайте https://widaxo.rocket.chat/home 

Чтобы восстановить пароль мне нужно:

- нажать на кнопку "Forgot your password?"
- ввести e-mail, на который был зарегистрирован пользователь
- нажать кнопку "Send instructions"
- перейти по специальной ссылке в письме на электронную почту

Результат: пароль изменен

### Сценарий 7: Поиск и доступ к каталогу рабочей области с помощью значка каталога

Чтобы воспользоваться каталогом, мне нужно:

Результат:  
- Пользователь получает доступ к каталогу рабочей области.
- Пользователь может выполнить поиск по каналам, пользователям и командам, используя ключевые слова или фразы.
- Результаты поиска отображаются пользователю, позволяя ему быстро найти нужные каналы, пользователей или команды.
- Пользователь может выбрать нужный канал, пользователя или команду из результатов поиска и перейти к ним для дальнейшего взаимодействия.
- Значок каталога предоставляет удобный способ доступа к каталогу и поиску внутри рабочей области, что экономит время и упрощает навигацию для пользователя.

### Сценарий 8: Настройка отображения боковой панели рабочей области с помощью значка отображения

Цель: настроить способ отображения чатов в боковой панели рабочей области

Дано: нахожусь в интерфейсе рабочей области и имею доступ к значку отображения

Чтобы совершить данное действие мне нужно:
- щелкнуть на значок отображения в интерфейсе боковой панели рабочей области
- открыть настройки отображения.
- настроить способ отображения, сортировку и группировку бесед

Результат: 
- Пользователь получает доступ к настройкам отображения боковой панели рабочей области.
- Пользователь может настроить способ отображения чатов в боковой панели, выбрав опции, такие как компактный вид или полный вид.
- Пользователь может настроить сортировку чатов по различным критериям, например, по дате, имени или активности.
- Пользователь может группировать чаты по категориям, таким как команды, проекты или личные чаты.
- Изменения, внесенные в настройки отображения, применяются к боковой панели рабочей области, изменяя способ представления чатов.

## Audit logs

### Сценарий 18: Использование журнала аудита логов (Message Auditing Log)

Как: пользователь в разделе "Audit logs" во вкладке "Audit"

Цель: Проверить историю использования панели аудита сообщений, включая информацию о том, кто выполнял аудиты и когда был произведен аудит определенной беседы.

Дано: нахожусь в разделе "Audit logs" во вкладке "Audit"

Чтобы провести аудит логов мне нужно:
- войти в раздел "Audit logs"
-  просмотреть лог-историю пользователей, которые получили доступ к панели аудита сообщений
- очистить поля начальной и конечной даты, чтобы получить всю историю журнала, от старых до самых новых аудитов. Или выбрать определенный временной промежуток, чтобы увидеть, кто аудировал сообщения в определенный период времени.
- журнал аудита возвращает таблицу с информацией, включающей имя пользователя, поиск, дату и время аудита, а также количество результатов поиска, возвращенных каждым аудитом.
- применять фильтры, которые были применены для каждого аудита, в зависимости от типа аудита.

Результат:
- Пользователь может проверить историю использования панели аудита сообщений, узнать, кто выполнял аудиты и когда, а также изучить информацию о поисковых запросах и результате каждого аудита с помощью журнала аудита сообщений.

### Сценарий 60: Удаление сообщений из канала

Как: пользователь на странице настроек канала

Цель: Очистить канал от ненужных сообщений и файлов

Дано: нахожусь на странице настроек канала

Чтобы удалить сообщения из канала мне нужно:
- перейти в настройки канала и найти функцию "Prune Messages"
- заполнить необходимые поля:
    - "Newer than" (Более новые, чем): Удалить все сообщения, опубликованные после указанной даты
    - "Older than" (Более старые, чем): Удалить все сообщения, опубликованные до указанной даты
    - "Users" (Пользователи): Указать пользователей, чьи сообщения следует удалить. Добавить имена пользователей, и их сообщения будут удалены
    - "Inclusive" (Включительно): Удалять все сообщения, которые были отправлены в указанное время, которое вы указали в полях "Newer than" и "Older than"
    - "Exclude pinned messages" (Исключить закрепленные сообщения): Не удалять закрепленные сообщения
    - "Only remove the attached files, keep messages" (Удалить только прикрепленные файлы, сохранить сообщения): Удалить файлы, но оставить сообщения

Результат:
- Выбранные сообщения и файлы будут удалены из канала. 
- Обратного восстановления удаленных сообщений и файлов не будет.

### Сценарий 212: Настройка параметров подключения LDAP

Как: пользователь в панели "Администрирование"

Цель: Установить параметры аутентификации, таймаутов и шифрования для подключения к серверу LDAP в Rocket.Chat

Дано: нахожусь во вкладке "LDAP"

Чтобы настроить параметры подключения LDAP мне нужно:
- перейти в раздел "LDAP Settings в административной панели
- в разделе "Connection на вкладке "Connection Settings" установите следующие параметры:
    - включить: Включает или отключает аутентификацию LDAP.
    - тип сервера: Позволяет выбрать тип сервера, с которым вы подключаетесь.
    - хост: Хост или доменное имя подключения LDAP, например, ldap.example.com или 10.0.0.30.
    - порт: Порт подключения LDAP, обычно 389 или 636.
    - параметры "Reconnect" и "Login Fallback" доступны только для рабочих пространств Enterprise Edition.
    - аутентификация:
       - включить: Включает или отключает аутентификацию для пропуска связывания с пользователем DN и паролем.
       - пользователь DN: Логин административной учетной записи или пользователя LDAP с доступом к активному каталогу.
       - пароль: Пароль для пользователя DN.
    - шифрование:
       - шифрование: Метод шифрования, используемый для защиты связи с сервером LDAP.
       - CA Cert: Сертификат центра сертификации (CA), используемый для подписи.
       - отклонить неподтвержденные: Отключите эту опцию, чтобы разрешить сертификаты, которые не могут быть проверены.
    - таймауты:
       - Timeout (ms): Время ожидания в миллисекундах после начала поиска перед возвратом ошибки.
       - Connection Timeout (ms): Максимальное время в миллисекундах до истечения таймаута подключения.
       - Idle Timeout (ms): Время ожидания в миллисекундах после последней операции LDAP до закрытия соединения.
- после завершения настроек сохранить изменения

Результат: 
- Параметры подключения LDAP установлены и сохранены успешно.

### Сценарий 213: Поиск пользователей в LDAP

Как: пользователь в панели "Администрирование"

Цель: Выполнить поиск пользователей LDAP с использованием заданных параметров

     - Base DN (Базовый DN): Полное квалифицированное имя раздела LDAP, в котором вы хотите производить поиск пользователей и групп. Пример: ou=Users+ou=Projects,dc=Example,dc=com

     - Search Size Limit (Лимит поиска): Максимальное количество записей, которые будут возвращены. Оно должно быть больше размера поиска (Search Size)

     - Group ObjectClass (Объектный класс группы): Идентификатор объектного класса группы
     - Group ID Attribute (Атрибут идентификатора группы): Атрибут идентификатора группы
     - Group Member Attribute (Атрибут участника группы): Атрибут участника группы
     - Group Member Format (Формат участника группы): Формат участника группы
     - Group name (Имя группы): Имя группы, к которой пользователь должен принадлежать

     - Advanced Sync (Расширенная синхронизация):
       - Sync User Active State (Синхронизация состояния активности пользователя): определить, должны ли пользователи быть включены или отключены в Rocket.Chat на основе состояния LDAP
       - Attributes to Query (Атрибуты для запроса): указать атрибуты, которые должны быть возвращены при запросах LDAP, разделяя их запятыми
       - Auto Logout Deactivated Users (Автоматический выход неактивных пользователей):
         - Enable Auto Logout (Включить автоматический выход): включить или отключить автоматический выход
         - Auto Logout Interval (Интервал автоматического выхода): установить интервал для автоматического выхода
       - Background Sync (Фоновая синхронизация):
         - Background Sync (Фоновая синхронизация): включить периодическую фоновую синхронизацию
         - Background Sync Interval (Интервал фоновой синхронизации): интервал между синхронизациями в формате Cron
         - Background Sync Import New Users (Синхронизация новых пользователей): импортировать всех пользователей из LDAP, которых еще нет в Rocket.Chat
         - Background Sync Update Existing Users (Обновление данных существующих пользователей): обновить аватар, поля, имя пользователя и т.д. у всех пользователей, уже импортированных из LDAP, на каждом интервале синхронизации
         - Avatar Background Sync (Фоновая синхронизация аватара): включить отдельный фоновый процесс синхронизации аватара пользователя
         - Avatar Background Sync Interval (Интервал фоновой синхронизации аватара): интервал между синхронизацией аватара в формате Cron
     - Sync Channels (Синхронизация каналов):
       - Auto Sync LDAP Groups to Channels (Автоматическая синхронизация групп LDAP с каналами): включить эту функцию, чтобы автоматически добавлять пользователей в канал на основе их группы LDAP
       - Channel Admin (Администратор канала): определяет пользователь, который станет администратором созданного автоматически канала во время синхронизации пользователей
       - User Group filter (Фильтр пользовательской группы): фильтр поиска LDAP, который проверяет, принадлежит ли пользователь к группе
       - LDAP Group BaseDN (Базовый DN группы LDAP): базовый DN группы LDAP
       - LDAP Group Channel Map (Карта группы LDAP для канала Rocket.Chat): сопоставление групп LDAP с каналами Rocket.Chat в формате JSON
     - Sync Custom Fields (Синхронизация пользовательских полей):
       - Sync Custom Fields (Синхронизация пользовательских полей): включить синхронизацию пользовательских полей
       - Custom Fields Mapping (Отображение пользовательских полей): карта пользовательского поля для синхронизации

    - Пользовательское сообщение при автоматическом закрытии комнаты из-за неактивности посетителя: введите пользовательское сообщение

    - Список разрешенных департаментов для пересылки (необязательно): позволяет установить ограниченный список департаментов, которые могут получать чаты из этого департамента
    - Резервный департамент для пересылки: выберите резервный департамент для пересылки
    - Запросить теги перед закрытием разговора: позволяет агенту установить теги для разговора для последующего поиска
    - Теги для закрытия разговора: как администратор, вы можете добавить теги разговора для этого департамента
    - Агенты: назначьте агентов на этот департамент

    - Нет департаментов: все агенты Live Chat будут получать входящие чаты по круговой схеме
    - Один департамент: все входящие чаты будут направляться только агентам этого департамента по круговой схеме
    - Более одного департамента: посетитель должен выбрать департамент, с которым он хочет связаться при регистрации

- после обновления приоритета, чат будет перемещен в соответствующую позицию в очереди, например, чат с приоритетом "highest" будет перенесен на верх очереди
- также можно изменить приоритет чата из раздела "Редактировать комнату" в информации о чате

Результат: 
- Приоритезация позволяет организовать очередь конверсаций в Omnichannel, что увеличивает эффективность обработки чатов и позволяет уделять больше внимания наиболее важным чатам. Также можно использовать фильтры по приоритетам для быстрого доступа к чатам с высоким приоритетом.
